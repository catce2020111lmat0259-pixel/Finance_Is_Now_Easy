# Sprints do Projeto

Este documento apresenta a evolução incremental do desenvolvimento do sistema, organizando as atividades realizadas em cada sprint.

---

## Sprint 1

**Período:** Março (início)

### Objetivo

Definição inicial do projeto.

### Atividades

* Inception do sistema
* Definição da ideia principal
* Identificação do problema
* Definição do público-alvo

### Entregas

* Descrição inicial do projeto

---

## Sprint 2

**Período:** Março / Abril

### Objetivo

Construção e organização dos requisitos do sistema.

### Atividades

* Criação do Product Backlog
* Escrita das Histórias de Usuário
* Definição de critérios de aceitação
* Identificação dos atores
* Definição das regras de negócio

### Entregas

* `backlog.md`
* `historias-de-usuario.md`
* `atores.md`
* `regras-de-negocio.md`

---

## Sprint 3

**Período:** Abril

### Objetivo

Prototipação inicial das funcionalidades.

### Atividades

* Desenvolvimento inicial das HUs:

  * HU01 – Receitas
  * HU02 – Despesas
  * HU04 – Categorias
  * HU09 – Metas
* Criação de fluxos iniciais das funcionalidades
* Primeiros testes de funcionamento

### Entregas

* Protótipos funcionais iniciais
* Primeiros fluxos documentados

---

## Sprint 4 (Atual)

**Período:** Abril / Maio

### Objetivo

Estruturação do sistema e implementação organizada com backend.

### Atividades

* Organização do repositório conforme padrão do professor
* Estruturação do backend com Flask (MVC)
* Implementação das funcionalidades:

  * HU04 – Categorias
  * HU09 – Metas
  * HU16 – Login
* Criação de documentação:

  * README
  * Modelagem
  * Fluxos das funcionalidades

### Entregas

* Backend estruturado (MVC)
* Funcionalidades em desenvolvimento
* Documentação organizada

---

## Sprint 5 (Planejada)

**Período:** Maio/ Junho

### Objetivo

Evoluir e consolidar as funcionalidades principais.

### Atividades

* Continuação da implementação das HUs principais
* Integração entre backend e templates
* Ajustes nas regras de negócio

### Entregas esperadas

* Funcionalidades principais operacionais
* Sistema parcialmente integrado

---

## Sprint 6 (Planejada)

**Período:** Junho

### Objetivo

Expandir funcionalidades e iniciar fase de testes.

### Atividades

* Implementação de funcionalidades complementares
* Melhorias de interface
* Testes iniciais do sistema

### Entregas esperadas

* Funcionalidades adicionais implementadas
* Sistema mais completo

---

## Sprint 7 (Planejada)

**Período:** Junho

### Objetivo

Finalização do sistema.

### Atividades

* Integração completa do sistema
* Testes finais
* Correção de bugs
* Preparação para entrega

### Entregas esperadas

* Sistema funcional completo
* Projeto pronto para apresentação

---

## Sprint 8 (Planejada)

**Período:** Final de Junho / Início de Julho

### Objetivo

Ajustes finais e consolidação do projeto para entrega definitiva.

### Atividades

* Revisão geral do sistema
* Ajustes finais identificados nos testes
* Organização final do repositório
* Atualização da documentação
* Preparação da apresentação final

### Entregas esperadas

* Projeto totalmente finalizado
* Documentação completa
* Repositório organizado conforme exigências
